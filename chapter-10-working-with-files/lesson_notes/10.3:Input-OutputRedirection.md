# **10.3 Input/Output Redirection**

## **Overview**

* **I/O Redirection** allows you to control **where input comes from** and **where output goes**.
* Instead of using the terminal (screen/keyboard), you can:

    * Send output to files
    * Read input from files

---

# **Understanding Standard Streams**

Linux uses three default communication channels called **streams**:

---

## **1. STDIN (Standard Input)**

* Stream **#0**
* Default source: **keyboard**
* Used when a command needs input from the user

### **Example**

```bash
cat
```

* Waits for user input
* Displays whatever you type

---

## **2. STDOUT (Standard Output)**

* Stream **#1**
* Default destination: **terminal (screen)**
* Contains **normal command output**

---

## **3. STDERR (Standard Error)**

* Stream **#2**
* Default destination: **terminal (screen)**
* Contains **error messages**

---

## **Key Idea**

By default:

* STDIN → keyboard
* STDOUT → screen
* STDERR → screen

👉 Redirection allows changing these defaults.

---

# **Redirection Symbols**

| Symbol | Purpose                         |
| ------ | ------------------------------- |
| `>`    | Redirect output (overwrite)     |
| `>>`   | Append output                   |
| `<`    | Redirect input                  |
| `2>`   | Redirect STDERR                 |
| `&>`   | Redirect both STDOUT and STDERR |

---

# **10.3.1 Redirecting STDOUT**

## **Default Behavior**

```bash
echo "Line 1"
```

Output appears on screen.

---

## **Redirect Output to File**

```bash
echo "Line 1" > example.txt
```

### **What Happens**

* Output is **saved in file**
* Nothing displayed on screen

---

## **View File Content**

```bash
cat example.txt
```

---

## **Overwrite Behavior ⚠️**

```bash
echo "New line 1" > example.txt
```

* Replaces existing content

---

## **Append Output**

```bash
echo "Another line" >> example.txt
```

### **Result**

* Adds text to **end of file**
* Existing content is preserved

---

# **10.3.2 Redirecting STDERR**

## **Example Error**

```bash
ls /fake
```

Output:

```
No such file or directory
```

---

## **Redirect STDOUT Only**

```bash
ls /fake > output.txt
```

* Error still appears on screen
* Confirms it is **STDERR**

---

## **Redirect STDERR**

```bash
ls /fake 2> error.txt
```

### **Explanation**

* `2>` specifies **error stream**
* Errors saved in `error.txt`

---

## **View Errors**

```bash
cat error.txt
```

---

# **10.3.3 Redirecting Multiple Streams**

## **Command Producing Both Outputs**

```bash
ls /fake /etc/ppp
```

* `/fake` → error (STDERR)
* `/etc/ppp` → valid output (STDOUT)

---

## **Redirect Only STDOUT**

```bash
ls /fake /etc/ppp > example.txt
```

* STDOUT → file
* STDERR → screen

---

## **Redirect Only STDERR**

```bash
ls /fake /etc/ppp 2> error.txt
```

* STDERR → file
* STDOUT → screen

---

## **Redirect Both STDOUT and STDERR**

```bash
ls /fake /etc/ppp &> all.txt
```

### **Explanation**

* `&>` = redirect **both streams**
* Combines:

    * `1>` (STDOUT)
    * `2>` (STDERR)

---

## **Important Behavior**

* Errors appear **first**
* Then normal output

---

## **Redirect to Different Files**

```bash
ls /fake /etc/ppp > output.txt 2> error.txt
```

### **Result**

* STDOUT → `output.txt`
* STDERR → `error.txt`

---

## **Order Does NOT Matter**

```bash
ls /fake /etc/ppp 2> error.txt > output.txt
```

* Same result

---

# **10.3.4 Redirecting STDIN**

## **Concept**

* Redirect **input source**
* Instead of keyboard → use a file

---

## **Basic Example with `cat`**

```bash
cat
```

* Takes input from keyboard

---

## **Stop the Command**

```bash
Ctrl + C
```

---

## **Using STDOUT Redirection to Create File**

```bash
cat > new.txt
```

Type:

```
Hello
How are you?
Goodbye
```

Then press:

```bash
Ctrl + D
```

---

## **View File**

```bash
cat new.txt
```

---

# **Using STDIN Redirection**

## **Example with `tr` Command**

* Converts characters

---

## **Without File**

```bash
tr 'a-z' 'A-Z'
```

* Reads from keyboard
* Converts to uppercase

---

## **Problem**

```bash
tr 'a-z' 'A-Z' example.txt
```

* ❌ Fails (does not accept file argument)

---

## **Solution: Redirect STDIN**

```bash
tr 'a-z' 'A-Z' < example.txt
```

### **What Happens**

* Input comes from file instead of keyboard
* Output goes to screen

---

## **Save Output to File**

```bash
tr 'a-z' 'A-Z' < example.txt > newexample.txt
```

---

## **View Result**

```bash
cat newexample.txt
```

---

# **Key Takeaways**

## **1. Stream Numbers**

| Stream | Name   | Number |
| ------ | ------ | ------ |
| STDIN  | Input  | 0      |
| STDOUT | Output | 1      |
| STDERR | Error  | 2      |

---

## **2. Most Common Usage**

* STDOUT redirection → **very common**
* STDERR redirection → **sometimes**
* STDIN redirection → **rare**

---

## **3. Important Rules**

* `>` overwrites
* `>>` appends
* `2>` handles errors
* `&>` handles both output and errors
* `<` redirects input

---

# **Practical Examples**

```bash
# Save output
ls > files.txt

# Append output
ls >> files.txt

# Save errors
ls /fake 2> errors.txt

# Save everything
ls /fake /etc &> all.txt

# Separate outputs
ls /fake /etc > output.txt 2> error.txt

# Input from file
tr 'a-z' 'A-Z' < input.txt

# Input + output redirection
tr 'a-z' 'A-Z' < input.txt > output.txt
```

---

# **Summary**

* I/O redirection allows **flexible control of data flow**
* You can:

    * Save output
    * Capture errors
    * Feed input from files
* Essential for:

    * Automation
    * Scripting
    * System administration
