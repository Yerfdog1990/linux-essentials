# **10.5 Viewing File Statistics**

## **Overview**

* The `wc` command (**word count**) is used to display **file statistics**.
* It provides information about:

    * Number of **lines**
    * Number of **words**
    * Number of **bytes (characters)**

---

## **Basic Syntax**

```bash id="t9m81g"
wc filename
```

---

## **Example**

```bash id="o0cq0u"
wc /etc/passwd /etc/passwd-
```

### **Output**

```id="z6v1kb"
35   56 1710 /etc/passwd
34   55 1665 /etc/passwd-
69  111 3375 total
```

---

## **Understanding the Output**

Each line contains **four columns**:

| Column | Meaning             |
| ------ | ------------------- |
| 1      | Number of **lines** |
| 2      | Number of **words** |
| 3      | Number of **bytes** |
| 4      | File name           |

---

## **Multiple Files**

* When more than one file is provided:

    * `wc` shows stats for each file
    * Then displays a **total summary**

---

# **Viewing Specific Statistics**

You can limit output using options:

---

## **1. Line Count (`-l`)**

```bash id="5y3l0q"
wc -l file.txt
```

* Shows only **number of lines**

---

## **2. Word Count (`-w`)**

```bash id="kwow2v"
wc -w file.txt
```

* Shows only **number of words**

---

## **3. Byte Count (`-c`)**

```bash id="u7h6lm"
wc -c file.txt
```

* Shows **number of bytes (characters)**

---

## **4. Combine Options**

```bash id="67x1tc"
wc -l -w file.txt
```

* Shows both **lines and words**

---

# **Using `wc` with Pipes**

## **Why Use Pipes?**

* Count results from **other commands**
* Useful for analyzing command output

---

## **Example: Count Files in Directory**

```bash id="4uq8bl"
ls /etc/ | wc -l
```

### **Explanation**

1. `ls /etc/` → lists files
2. `wc -l` → counts lines (each file = one line)

### **Result**

```id="0c2o6m"
142
```

👉 This means there are **142 files/directories** in `/etc`

---

# **Key Concepts**

## **1. What is a Byte?**

* 1 byte = **1 character** (in text files)

---

## **2. Default Behavior**

* `wc` shows:

    * Lines
    * Words
    * Bytes

---

## **3. Flexibility**

* Can work with:

    * Files
    * Output from other commands (via pipes)

---

# **Practical Examples**

```bash id="pvawqk"
# Count lines, words, and bytes
wc file.txt

# Count only lines
wc -l file.txt

# Count only words
wc -w file.txt

# Count only characters
wc -c file.txt

# Count lines in multiple files
wc -l file1.txt file2.txt

# Count number of files in directory
ls | wc -l

# Count number of users (lines in passwd file)
wc -l /etc/passwd
```
