# **10.7 Filter File Contents**

## **Overview**

* The `grep` command is used to **search and filter text**.
* It returns **lines that match a specific pattern**.

---

## **Key Idea**

* Input can come from:

    * Files
    * Output of other commands (via pipes)
* Output is:

    * Only **matching lines**

---

## **Basic Syntax**

```bash id="j3w5nl"
grep [options] pattern file
```

---

# **Basic Example**

```bash id="p7ojjw"
grep bash /etc/passwd
```

### **Output**

```id="kz9s9y"
root:x:0:0:root:/root:/bin/bash
sysadmin:x:1001:1001:System Administrator,,,,:/home/sysadmin:/bin/bash
```

---

## **Explanation**

* Searches for the pattern `"bash"`
* Displays lines containing that pattern

---

# **Pattern Matching**

## **Patterns Can Be**

* Simple text (e.g., `bash`)
* Complex patterns (using **regular expressions**)

---

# **Highlight Matches**

## **Option: `--color`**

```bash id="k7a4g4"
grep --color bash /etc/passwd
```

* Highlights matched text (usually in red)
* Often enabled automatically via alias

---

# **Counting Matches**

## **Option: `-c`**

```bash id="q3a3xr"
grep -c bash /etc/passwd
```

### **Output**

```id="5b9vso"
2
```

### **Explanation**

* Counts number of matching lines
* Does **not display the lines**

---

# **Show Line Numbers**

## **Option: `-n`**

```bash id="0yrp9f"
grep -n bash /etc/passwd
```

### **Output**

```id="ew7b2l"
1:root:x:0:0:root:/root:/bin/bash
27:sysadmin:x:1001:1001:System Administrator,,,,:/home/sysadmin:/bin/bash
```

### **Explanation**

* Displays:

    * Line number
    * Matching content

---

# **Invert Matches**

## **Option: `-v`**

```bash id="j6q6k4"
grep -v nologin /etc/passwd
```

### **Explanation**

* Shows lines **NOT containing** the pattern

---

# **Case-Insensitive Search**

## **Option: `-i`**

```bash id="jph0ne"
grep -i the newhome.txt
```

### **Explanation**

* Ignores capitalization
* Matches:

    * `the`, `The`, `THE`, etc.

---

# **Match Whole Words Only**

## **Option: `-w`**

```bash id="hqb0zj"
grep -w are newhome.txt
```

---

## **Example Comparison**

### Without `-w`

```bash id="2wq36f"
grep are newhome.txt
```

### Output:

```id="avz7nb"
There are three bathrooms.
Beware of the ghost in the bedroom.
```

---

### With `-w`

```bash id="o3nk7k"
grep -w are newhome.txt
```

### Output:

```id="4z6y7r"
There are three bathrooms.
```

---

## **Explanation**

* Without `-w`:

    * Matches **partial words** (e.g., "Beware")
* With `-w`:

    * Matches only **whole words**

---

# **Using `grep` with Pipes**

## **Example**

```bash id="u6d8y0"
ls /etc | grep conf
```

* Filters output of `ls`
* Shows only files containing `"conf"`

---

## **Another Example**

```bash id="ntd2qa"
cat /etc/passwd | grep bash
```

* Equivalent to:

```bash id="hivx3x"
grep bash /etc/passwd
```

---

# **Key Options Summary**

| Option    | Function          |
| --------- | ----------------- |
| `--color` | Highlight matches |
| `-c`      | Count matches     |
| `-n`      | Show line numbers |
| `-v`      | Invert match      |
| `-i`      | Ignore case       |
| `-w`      | Match whole words |

---

# **Key Concepts**

## **1. Line-Based Filtering**

* `grep` returns **entire lines**, not just matching text

---

## **2. Pattern Matching**

* Can be:

    * Simple text
    * Complex patterns (regex)

---

## **3. Flexible Input**

* Works with:

    * Files
    * Pipes (command output)

---

# **Practical Examples**

```bash id="v4yzjs"
# Find users with bash shell
grep bash /etc/passwd

# Count matches
grep -c bash /etc/passwd

# Show line numbers
grep -n bash /etc/passwd

# Ignore case
grep -i error logfile.txt

# Exclude matches
grep -v nologin /etc/passwd

# Match whole words only
grep -w root /etc/passwd

# Use with pipe
ls | grep txt
```

---

# **Common Use Cases**

## **1. System Administration**

```bash id="kk4jht"
grep root /etc/passwd
```

---

## **2. Log Analysis**

```bash id="72gq1m"
grep error /var/log/syslog
```

---

## **3. Searching Files**

```bash id="g9s2d9"
grep keyword file.txt
```

---

# **Summary**

## **What `grep` Does**

* Filters lines based on **matching patterns**

---

## **Core Strengths**

* Fast and powerful
* Works with files and pipes
* Supports advanced pattern matching

---

## **Important Tips**

* Use `-i` for case-insensitive searches
* Use `-w` to avoid partial matches
* Use `-v` to exclude results
* Combine with pipes for powerful workflows
