# **10.6 Filter File Sections**

## **Overview**

* The `cut` command is used to **extract specific parts (columns/sections)** of text.
* Works on:

    * Files
    * Standard input (via pipes)

---

## **Key Use Case**

* Useful for working with **structured or delimited data**

    * e.g., system files like `/etc/passwd`
    * CSV (comma-separated values)
    * Logs and command output

---

# **Understanding Delimited Files**

## **What is a Delimiter?**

* A **delimiter** is a character that separates fields in a line

### **Common Delimiters**

| Delimiter | Example           |
| --------- | ----------------- |
| `:`       | `/etc/passwd`     |
| `,`       | CSV files         |
| Tab       | Default for `cut` |

---

## **Basic Syntax**

```bash
cut [options] file
```

---

# **Important Options**

## **1. `-d` (Delimiter)**

* Specifies the **field separator**

```bash
cut -d: file
```

---

## **2. `-f` (Fields)**

* Specifies **which fields to extract**

### **Formats**

* Single field → `-f1`
* Multiple fields → `-f1,3`
* Range → `-f2-5`

---

# **Example: Extract Fields**

## **Command**

```bash
cut -d: -f1,5-7 mypasswd
```

---

## **Explanation**

* `-d:` → use colon as delimiter
* `-f1,5-7` → extract:

    * Field 1
    * Fields 5 to 7

---

## **Output**

```bash
root:root:/root:/bin/bash
daemon:daemon:/usr/sbin:/usr/sbin/nologin
bin:bin:/bin:/usr/sbin/nologin
sys:sys:/dev:/usr/sbin/nologin
sync:sync:/bin:/bin/sync
```

---

## **What Was Extracted**

| Field | Meaning (in `/etc/passwd`) |
| ----- | -------------------------- |
| 1     | Username                   |
| 5     | Description                |
| 6     | Home directory             |
| 7     | Shell                      |

---

# **Extracting by Character Position**

## **Option: `-c`**

* Extract based on **character positions**, not fields

---

## **Syntax**

```bash
cut -c positions
```

### **Formats**

* Single position → `-c1`
* Range → `-c1-10`
* Multiple → `-c1-5,10-15`

---

# **Example: Using `ls -l` Output**

```bash
ls -l | cut -c1-11,50-
```

---

## **Explanation**

* `-c1-11` → file type + permissions + space
* `-c50-` → filename (starting at position 50)

---

## **Output**

```bash
total 44
drwxr-xr-x Desktop
drwxr-xr-x Documents
drwxr-xr-x Downloads
drwxr-xr-x Music
drwxr-xr-x Pictures
drwxr-xr-x Public
drwxr-xr-x Templates
drwxr-xr-x Videos
-rw-rw-r-- all.txt
-rw-rw-r-- example.txt
-rw-rw-r-- mypasswd
-rw-rw-r-- new.txt
```

---

## **Why This Works**

* `ls -l` output has **fixed column positions**
* Makes it suitable for **character-based extraction**

---

# **Using `cut` with Pipes**

## **Example**

```bash
cat mypasswd | cut -d: -f1
```

* Extracts only **usernames**

---

## **Another Example**

```bash
ls -l | cut -c1-10
```

* Extracts only **permissions**

---

# **Key Concepts**

## **1. Field-Based vs Character-Based**

| Method          | Option | Use Case         |
| --------------- | ------ | ---------------- |
| Field-based     | `-f`   | Delimited files  |
| Character-based | `-c`   | Fixed-width data |

---

## **2. Default Behavior**

* Assumes **tab delimiter**
* Use `-d` to override

---

## **3. Flexible Selection**

* Single field
* Multiple fields
* Ranges

---

# **Practical Examples**

```bash
# Extract usernames
cut -d: -f1 /etc/passwd

# Extract home directories
cut -d: -f6 /etc/passwd

# Extract multiple fields
cut -d: -f1,6 /etc/passwd

# Extract range of fields
cut -d: -f2-4 file.txt

# Extract characters
cut -c1-5 file.txt

# Use with pipe
ls -l | cut -c1-10
```

---

# **Common Use Cases**

## **1. System Administration**

* Extract user info:

```bash
cut -d: -f1 /etc/passwd
```

---

## **2. Data Processing**

* Work with CSV files:

```bash
cut -d, -f2 data.csv
```

---

## **3. Log Analysis**

* Extract specific columns from logs

---

# **Summary**

## **What `cut` Does**

* Extracts **specific parts of text**

---

## **Core Options**

| Option | Function          |
| ------ | ----------------- |
| `-d`   | Set delimiter     |
| `-f`   | Select fields     |
| `-c`   | Select characters |

---

## **Key Advantages**

* Fast and simple
* Works with pipes
* Ideal for structured data

---

## **Important Tip**

> Use `cut` when you only need **specific columns**, not the entire file.
