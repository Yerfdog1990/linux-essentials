# **10.4 Sorting Files or Input**

## **Overview**

* The `sort` command is used to **rearrange lines** in:

    * Files
    * Command output (via pipes)
* Sorting can be done:

    * **Alphabetically (dictionary order)** *(default)*
    * **Numerically**

---

## **Basic Usage**

```bash
sort filename
```

---

## **Example: Creating a File**

```bash
head -5 /etc/passwd > mypasswd
cat mypasswd
```

### **Sample Content**

```
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
sync:x:4:65534:sync:/bin:/bin/sync
```

---

## **Sorting the File**

```bash
sort mypasswd
```

### **Result**

```
bin:x:2:2:bin:/bin:/usr/sbin/nologin
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
root:x:0:0:root:/root:/bin/bash
sync:x:4:65534:sync:/bin:/bin/sync
sys:x:3:3:sys:/dev:/usr/sbin/nologin
```

### **Explanation**

* Sorted **alphabetically**
* Based on the **entire line**

---

# **10.4.1 Fields and Sort Options**

## **What Are Fields?**

* A **field** is a section of a line
* Fields are separated by a **delimiter**

### **Default Delimiter**

* Whitespace (spaces/tabs)

### **Custom Delimiters**

* Example:

    * `:` (colon)
    * `,` (comma)

---

# **Important `sort` Options**

## **1. `-t` (Field Delimiter)**

* Specifies **what separates fields**

```bash
sort -t: filename
```

---

## **2. `-k` (Field Selection)**

* Specifies **which field to sort by**
* Fields start at **1**

```bash
sort -k3 filename
```

---

## **3. `-n` (Numeric Sort)**

* Sorts numbers **numerically**
* Without it → numbers are treated as text

```bash
sort -n filename
```

---

## **Example: Sorting by Third Field (Numeric)**

```bash
sort -t: -n -k3 mypasswd
```

### **Explanation**

* `-t:` → colon delimiter
* `-k3` → sort by 3rd field
* `-n` → numeric sort

---

### **Result**

```
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
sync:x:4:65534:sync:/bin:/bin/sync
```

---

# **Reverse Sorting**

## **Option: `-r`**

* Reverses sort order

```bash
sort -t: -n -r -k3 mypasswd
```

### **Result**

```
sync:x:4:65534:sync:/bin:/bin/sync
sys:x:3:3:sys:/dev:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
root:x:0:0:root:/root:/bin/bash
```

---

# **Sorting Multiple Fields**

## **Why Use Multiple Fields?**

* When:

    * Primary fields are identical
    * Need secondary sorting criteria

---

## **Example File (CSV)**

```bash
cat os.csv
```

### **Content**

```
1970,Unix,Richie
1987,Minix,Tanenbaum
1970,Unix,Thompson
1991,Linux,Torvalds
```

---

## **Sort by Multiple Fields**

```bash
sort -t, -k2 -k1n -k3 os.csv
```

---

## **Explanation of Options**

| Option | Meaning                          |
| ------ | -------------------------------- |
| `-t,`  | Use comma as delimiter           |
| `-k2`  | Sort by field #2 (OS name)       |
| `-k1n` | Then by field #1 (year, numeric) |
| `-k3`  | Then by field #3 (name)          |

---

## **Result**

```
1991,Linux,Torvalds
1987,Minix,Tanenbaum
1970,Unix,Richie
1970,Unix,Thompson
```

---

## **Sorting Logic**

1. **Primary sort** → OS name
2. **Secondary sort** → Year
3. **Tertiary sort** → Last name

---

# **Using `sort` with Pipes**

## **Example**

```bash
ls /etc | sort
```

* Sorts command output alphabetically

---

## **Combined Example**

```bash
ls /etc | sort | head
```

* Sort → then show first 10 results

---

# **Key Concepts**

## **1. Default Behavior**

* Sorts **entire lines alphabetically**

---

## **2. Numeric vs Alphabetic**

* Without `-n`:

    * `"10"` comes before `"2"`
* With `-n`:

    * `2` comes before `10`

---

## **3. Field-Based Sorting**

* Use:

    * `-t` → delimiter
    * `-k` → field

---

## **4. Multi-Level Sorting**

* Multiple `-k` options = multiple priorities

---

# **Practical Examples**

```bash
# Basic sort
sort file.txt

# Numeric sort
sort -n numbers.txt

# Reverse sort
sort -r file.txt

# Sort by field (colon-separated)
sort -t: -k3 file.txt

# Numeric + reverse
sort -t: -n -r -k3 file.txt

# Multiple fields
sort -t, -k2 -k1n -k3 data.csv

# Sort command output
ls | sort
```

---

# **Summary**

## **What `sort` Does**

* Organizes text data:

    * Alphabetically
    * Numerically
    * By specific fields

---

## **Core Options**

| Option | Purpose       |
| ------ | ------------- |
| `-t`   | Set delimiter |
| `-k`   | Choose field  |
| `-n`   | Numeric sort  |
| `-r`   | Reverse order |

---

## **Best Practices**

* Always check:

    * **Delimiter**
    * **Field number**
    * **Data type (text vs number)**
* Use pipes to sort **command output**
* Use multiple `-k` for complex sorting
