# **10.8 Basic Regular Expressions**

## **Overview**

* **Regular Expressions (regex)** are patterns used to **search for text**.
* Used with commands like:

    * `grep`
    * `less`
    * `more`

---

## **Types of Characters**

### **1. Normal Characters**

* Match themselves
* Example:

    * `a` â†’ matches `a`

---

### **2. Special Characters**

* Have **special meanings**
* Used to define **patterns**

---

## **Types of Regex**

| Type                               | Description                   |
| ---------------------------------- | ----------------------------- |
| Basic Regular Expressions (BRE)    | Supported by most commands    |
| Extended Regular Expressions (ERE) | More advanced (use `grep -E`) |

---

## **Best Practice**

```bash id="r5e6b1"
grep 'pattern' file
```

* Use **single quotes `' '`**
* Prevents shell from interpreting special characters

---

# **Basic Regular Expression Characters**

| Character | Meaning                                   |
| --------- | ----------------------------------------- |
| `.`       | Any single character                      |
| `[ ]`     | Match one character from a list/range     |
| `[^ ]`    | Match any character NOT in list           |
| `*`       | Repeat previous character 0 or more times |
| `^`       | Beginning of line                         |
| `$`       | End of line                               |

---

# **10.8.1 The Period `.` Character**

## **Meaning**

* Matches **any single character** (except newline)

---

## **Example**

```bash id="go6p7g"
grep 'r..f' red.txt
```

### **Matches**

```id="r8bzcb"
reef
roof
```

### **Explanation**

* `r` â†’ first letter
* `..` â†’ any two characters
* `f` â†’ last letter

---

## **Another Example**

```bash id="qz7f9v"
grep '....' red.txt
```

* Matches words with **at least 4 characters**

---

# **10.8.2 The Bracket `[ ]` Characters**

## **Meaning**

* Match **one character** from a set or range

---

## **Examples**

### **Match Digits**

```bash id="6c4k8f"
grep '[0-9]' profile.txt
```

### **Output**

* Lines containing **numbers**

---

## **Ranges**

* `[a-z]` â†’ lowercase letters
* `[A-Z]` â†’ uppercase letters
* `[0-9]` â†’ digits

---

## **Invalid Range**

```bash id="slm8zq"

```

* âťŚ Error (range must be in correct order)

---

## **Negation with `^`**

```bash id="4p8m2k"
grep '[^0-9]' profile.txt
```

### **Meaning**

* Match lines containing **non-numeric characters**

* Does NOT mean â€śno numbersâ€ť

# **10.8.3 The Asterisk `*` Character**

## **Meaning**

* Matches **zero or more occurrences** of the previous character

---

## **Example**

```bash id="p0v8yx"
grep 're*d' red.txt
```

### **Matches**

```id="z3nbp7"
red
reeed
rd
reed
```

---

## **Using with Character Sets**

```bash id="w6tnkq"
grep 'r[oe]*d' red.txt
```

* Matches:

    * `r` + any number of `o` or `e` + `d`

---

## **Important Behavior âš ď¸Ź**

```bash id="1v7qlc"
grep 'z*' red.txt
```

* Matches **every line**
* Because `*` allows **zero occurrences**

---

## **Better Pattern**

```bash id="d7y0xb"
grep 'ee*' red.txt
```

* Matches lines with **at least one `e`**

---

# **10.8.4 Anchor Characters**

## **Purpose**

* Restrict match to:

    * Start of line
    * End of line

---

## **1. Beginning Anchor `^`**

```bash id="p7r9xm"
grep '^root' /etc/passwd
```

### **Matches**

* Lines **starting with `root`**

---

## **2. Ending Anchor `$`**

```bash id="yxh3pw"
grep 'r$' alpha-first.txt
```

### **Matches**

```id="c8u1dz"
B is for Bear
F is for Flower
```

---

## **Important**

* `^` â†’ must be at **start of pattern**
* `$` â†’ must be at **end of pattern**

---

# **10.8.5 The Backslash `\` Character**

## **Purpose**

* Escape special characters
* Treat them as **literal characters**

---

## **Example**

```bash id="7q9j0w"
grep 're\*' newhome.txt
```

### **Explanation**

* Searches for **literal `*`**
* Without `\`, `*` acts as regex operator

---

# **10.8.6 Extended Regular Expressions (ERE)**

## **Usage**

```bash id="o3k1dj"
grep -E 'pattern' file
```

---

## **Special Characters**

| Character | Meaning                                    |             |
| --------- | ------------------------------------------ | ----------- |
| `?`       | Previous character appears 0 or 1 time     |             |
| `+`       | Previous character appears 1 or more times |             |
| `         | `                                          | OR operator |

---

## **Examples**

### **Optional Character**

```bash id="5qf3sd"
grep -E 'colou?r' spelling.txt
```

* Matches:

    * `color`
    * `colour`

---

### **One or More**

```bash id="z6s9qa"
grep -E 'e+' red.txt
```

* Matches lines with **one or more `e`**

---

### **OR Operator**

```bash id="y3s8df"
grep -E 'gray|grey' spelling.txt
```

* Matches:

    * `gray`
    * `grey`

---

# **Key Concepts**

## **1. Pattern Matching**

* Regex allows **flexible searching**
* Can match:

    * Exact text
    * Complex patterns

---

## **2. Grep Integration**

* `grep` is the most common tool for regex

---

## **3. Anchoring**

* Use `^` and `$` to **narrow results**

---

## **4. Character Matching**

* `.` â†’ any character
* `[ ]` â†’ specific characters
* `[^ ]` â†’ exclude characters

---

## **5. Repetition**

* `*` â†’ zero or more
* `+` â†’ one or more
* `?` â†’ optional

---

# **Practical Examples**

```bash id="x0x6vu"
# Any 3-letter pattern starting with r and ending with t
grep 'r..t' file.txt

# Lines starting with root
grep '^root' /etc/passwd

# Lines ending with .txt
grep '\.txt$' filelist.txt

# Lines with digits
grep '[0-9]' file.txt

# Lines without digits
grep '[^0-9]' file.txt

# Match multiple options
grep -E 'cat|dog' animals.txt
```
