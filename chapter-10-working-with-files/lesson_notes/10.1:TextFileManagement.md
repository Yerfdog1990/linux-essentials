# **10.1 Introduction**

## **Overview**

* A large number of files in a Linux system are **text files**.
* **Text files**:

    * Contain **plain text only**
    * Do **not include formatting** (unlike Word documents)

### **Why Text Files Matter**

* Used for:

    * Configuration files (e.g., `/etc` directory)
    * Logs
    * Scripts
* Because they are so common, Linux provides **many commands** to:

    * View text
    * Modify text
    * Search within text

---

## **Shell Output Control (Redirection)**

Linux allows you to **control where command output goes**:

* Output can be:

    * Displayed on the terminal (default)
    * Saved to a file
    * Passed into another command

### **Basic Redirection Examples**

```bash
# Redirect output to a file (overwrite)
cat file.txt > output.txt

# Append output to a file
cat file.txt >> output.txt

# Use output as input for another command (pipe)
cat file.txt | less
```

👉 These features make Linux **flexible and powerful** for handling text.

---

# **10.1.1 Viewing Files in the Terminal**

## **The `cat` Command**

* Short for **concatenate**
* Used to:

    * Display file contents
    * Create files
    * Combine files

### **Basic Usage**

```bash
cat filename
```

### **Example**

```bash
cd Documents
cat food.txt
```

**Output:**

```
Food is good.
```

### **Other Uses**

```bash
# Combine files
cat file1.txt file2.txt > combined.txt

# Create a file (type content, then Ctrl+D to save)
cat > newfile.txt
```

### **Limitation of `cat`**

* Displays the **entire file at once**
* Not suitable for **large files**

---

# **10.1.2 Viewing Files Using a Pager**

## **Why Use a Pager?**

* `cat` is not ideal for large files
* Pager commands:

    * Show **one screen at a time**
    * Allow navigation (scrolling)

---

## **Common Pager Commands**

### **1. `less` (Recommended)**

* Advanced and feature-rich
* Default pager for commands like:

```bash
man ls
```

### **2. `more`**

* Older and simpler
* Always available
* Fewer features than `less`

---

## **Using `less`**

```bash
less filename
```

### **Example**

```bash
less words
```

---

# **10.1.2.1 Pager Movement Commands**

### **Essential Keys (Work in both `more` and `less`)**

| Key      | Action               |
| -------- | -------------------- |
| Spacebar | Move forward a page  |
| B        | Move backward a page |
| Enter    | Move forward a line  |
| Q        | Quit                 |
| H        | Help                 |

### **Important Notes**

* Press **H** inside `less` to see full help
* You don’t need to memorize all commands

---

# **10.1.2.2 Pager Searching Commands**

## **Search Forward**

```bash
/frog
```

* Press `/`
* Type search term
* Press **Enter**

### Behavior:

* Moves to **first match**
* Highlights all matches

---

## **Search Backward**

```bash
?frog
```

* Press `?`
* Searches **backward**

---

## **Navigate Search Results**

| Key       | Action         |
| --------- | -------------- |
| n         | Next match     |
| Shift + N | Previous match |

---

## **If No Match Found**

```
Pattern not found
```

---

## **Important Concept**

* Searches use **regular expressions (regex)**
* Allows pattern-based searching (covered later)

# **10.1.3 Head and Tail**

## **Purpose**

* Display **specific parts** of a file:

    * `head` → beginning
    * `tail` → end

---

## **Default Behavior**

* Both show **10 lines** by default

---

## **Using `head`**

```bash
head filename
```

### Example:

```bash
head /etc/sysctl.conf
```

---

## **Using `tail`**

```bash
tail filename
```

### Example:

```bash
tail /etc/sysctl.conf
```

---

## **Specifying Number of Lines**

### **Using `-n` option**

```bash
head -n 3 file.txt
tail -n 5 file.txt
```

### **Using shorthand**

```bash
head -3 file.txt
tail -5 file.txt
```

---

# **Special Options**

## **1. Negative Values (GNU Behavior)**

### **head**

```bash
head -n -3 file.txt
```

* Shows **all except last 3 lines**

### **tail**

```bash
tail -n -3 file.txt
```

* Shows **last 3 lines**

---

## **2. Positive Values with `+` (tail only)**

```bash
tail -n +25 file.txt
```

* Starts from **line 25 → end of file**

### Example:

```bash
nl /etc/passwd | tail -n +25
```

---

# **3. Real-Time File Monitoring**

## **Using `tail -f`**

```bash
tail -f filename
```

### Example:

```bash
tail -f /var/log/mail.log
```

### Use Case:

* Monitor logs in real-time
* Troubleshoot systems

---

## **How It Works**

* Continuously updates output as file changes
* Press **Ctrl + C** to stop

## **Key Commands**

| Command | Purpose                        |
| ------- | ------------------------------ |
| `cat`   | Display/concatenate files      |
| `less`  | View large files interactively |
| `more`  | Basic pager                    |
| `head`  | View beginning of file         |

---

## **Key Concepts**

* Linux heavily relies on **text files**
* Many tools exist to **view and manipulate text**
* **Redirection and pipes** increase flexibility
* Use:

    * `cat` → small files
    * `less` → large files
    * `head/tail` → partial viewing
    * `tail -f` → live monitoring
