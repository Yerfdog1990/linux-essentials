# **10.2 Command Line Pipes**

## **Overview**

* The **pipe (`|`)** is used to **connect commands together**.
* It sends the **output of one command** directly as **input to another command**.

### **Key Idea**

Instead of:

* Output going to the **terminal (screen)**

You can:

* Pass it to another command for **further processing**

---

## **Basic Syntax**

```bash
command1 | command2
```

### **How It Works**

1. `command1` runs
2. Its **output is NOT shown on screen**
3. Output is passed to `command2`
4. `command2` processes it and displays the result

---

## **Why Pipes Are Useful**

* Help **refine large outputs**
* Allow **combining multiple tools**
* Avoid creating temporary files
* Enable powerful **data processing workflows**

---

# **Using Pipes with Commands**

## **Example: Large Output Problem**

```bash
ls /etc
```

* Displays a **very long list**
* Output may be **cut off** in the terminal

---

## **Solution: Use Pipe with `head`**

```bash
ls /etc | head
```

### **What Happens**

* `ls /etc` → lists all files
* Output is passed to `head`
* `head` shows only **first 10 lines**

---

## **Key Concept**

* `head` is now working on **output**, not a file

👉 Pipes allow commands like `head` and `tail` to work on:

* Files ✅
* Output of other commands ✅

---

# **Multiple Pipes (Command Chaining)**

## **Syntax**

```bash
command1 | command2 | command3
```

### **Execution Flow**

1. Output of `command1` → input of `command2`
2. Output of `command2` → input of `command3`
3. Final output → displayed

---

## **Example with Multiple Pipes**

```bash
ls /etc/ssh | nl | tail -5
```

### **Step-by-Step**

1. `ls /etc/ssh` → list files
2. `nl` → add line numbers
3. `tail -5` → show last 5 lines

### **Output**

```bash
6  ssh_host_ed25519_key.pub
7  ssh_host_rsa_key
8  ssh_host_rsa_key.pub
9  ssh_import_id
10 sshd_config
```

---

# **Order of Pipes Matters ⚠️**

The **order of commands changes the result** because:

* Each command only sees the **output of the previous one**

---

## **Example 1**

```bash
ls /etc/ssh | nl | tail -5
```

### **Explanation**

* First: number ALL lines
* Then: show last 5 lines

👉 Output line numbers reflect original positions (e.g., 6–10)

---

## **Example 2**

```bash
ls /etc/ssh | tail -5 | nl
```

### **Explanation**

* First: get last 5 lines
* Then: number them

👉 Output line numbers are **1–5**

---

## **Key Difference**

| Command Order | Result                   |
| ------------- | ------------------------ |
| `nl → tail`   | Keeps original numbering |
| `tail → nl`   | Renumbers from 1         |

---

## **Important Principle**

> Each command only processes what it receives from the previous command.

---

# **Using Pipes Effectively**

## **1. Filtering Data**

```bash
ls /etc | head
```

* Get only the **top results**

---

## **2. Viewing Specific Sections**

```bash
ls /etc | tail -5
```

* Get **last 5 entries**

---

## **3. Adding Structure**

```bash
ls /etc | nl
```

* Add line numbers for readability

---

## **4. Combining Multiple Operations**

```bash
ls /etc | nl | head -10
```

* List files
* Number them
* Show first 10

---

# **Best Practices**

## **1. Think in Steps**

Break pipelines into stages:

1. Generate output
2. Filter it
3. Format it

---

## **2. Order Carefully**

* Changing order changes meaning
* Always ask:

    * “What is this command receiving as input?”

---

## **3. Keep It Simple**

* Start with one pipe
* Add more only when needed

---

# **Real-World Use Cases**

## **1. Analyzing Directories**

```bash
ls /var/log | head
```

## **2. Monitoring Logs**

```bash
tail -f /var/log/syslog | less
```

## **3. Structured Viewing**

```bash
ls /etc | nl | less
```

# **Summary**

## **Key Concepts**

* Pipe (`|`) connects commands
* Output of one command becomes input to another
* Enables **powerful command combinations**

---

## **Key Rules**

* Commands execute **left → right**
* Each command only sees **previous output**
* **Order matters**

---

## **Core Benefits**

* Efficient data processing
* Cleaner workflows
* No need for intermediate files

---

ls /etc | head
