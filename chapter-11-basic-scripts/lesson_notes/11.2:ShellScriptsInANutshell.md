## 11.2 Shell Scripts in a Nutshell

### 1. What is a Shell Script?

A **shell script** is a plain text file that contains a sequence of commands that the shell can execute. Instead of typing commands one by one in the terminal, you can store them in a file and run them all at once.

* Each line in the script represents a command.
* When executed, the shell reads the file and runs the commands in order.
* Shell scripts support **logic**, meaning they can make decisions (e.g., check if a file exists or evaluate output).

👉 This makes scripts powerful tools for **automation** and **efficiency**.

---

### 2. Why Use Shell Scripts?

Shell scripts are mainly used to:

* **Automate repetitive tasks**
* **Save time and effort**
* **Ensure consistency** (same steps executed exactly the same way every time)

📌 Example:
If you run the same five commands daily, you can combine them into one script and execute everything with a single command.

---

### 3. A Simple Shell Script Example

A script can be extremely simple—even just one command:

```bash
echo "Hello, World!"
```

If saved in a file called `test.sh`, running it will output:

```
Hello, World!
```

---

### 4. How to Run a Shell Script

There are two main ways to run a script:

#### a) Run with a Shell Interpreter

```bash
sh test.sh
```

* The script is passed as an argument to the shell (`sh`).
* No special permissions are required.

#### b) Run Directly

```bash
./test.sh
```

* The `./` means “run from the current directory.”
* This is needed because the current directory is usually **not in the `$PATH`**.

---

### 5. File Permissions and Executable Scripts

If you try to run a script directly and see:

```
Permission denied
```

👉 This means the script is **not executable**.

To fix it:

```bash
chmod +x test.sh
```

* `chmod` changes file permissions.
* `+x` adds execute permission.

After that:

```bash
./test.sh
```

will work correctly.

---

### 6. The Shebang (#!)

For more complex scripts, you often specify which shell should interpret the script using a **shebang**.

#### Examples:

```bash
#!/bin/sh
echo "Hello, World!"
```

or

```bash
#!/bin/bash
echo "Hello, World!"
```

---

### 7. What is a Shebang?

* The first line starting with `#!` is called a **shebang** (pronounced *sheh-bang*).
* It tells the system **which interpreter** to use to run the script.

💡 Key points:

* `#` = hash
* `!` = bang
* Together → “shebang”

---

### 8. How Shebang Works

* When you run a script **directly (`./script.sh`)**, the system uses the interpreter specified in the shebang.
* When you run:

  ```bash
  sh script.sh
  ```

  or

  ```bash
  bash script.sh
  ```

  👉 The specified shell is used **regardless of the shebang**.

---

### 9. Shebang Beyond Shell Scripts

The shebang is not limited to shell scripts. It is also used in other scripting languages such as:

* Perl
* Ruby
* Python

Any executable text file can use a shebang to define its interpreter.

---

### 10. Importance of Text Editors

Shell scripts must be written in **plain text**.

✅ Suitable tools:

* Terminal-based editors (e.g., `nano`, `vim`)
* Simple text editors

❌ Not suitable:

* Office tools like LibreOffice

👉 Reason:

* These tools add formatting and metadata, which breaks script execution.

---

### 11. Key Takeaways

* A shell script is a **file of executable commands**.
* Scripts allow **automation, consistency, and efficiency**.
* You can run scripts:

    * With a shell (`sh script.sh`)
    * Directly (`./script.sh`) after making them executable
* Use `chmod +x` to fix permission issues.
* The **shebang (`#!`)** defines which interpreter runs the script.
* Always write scripts in **plain text format**.
