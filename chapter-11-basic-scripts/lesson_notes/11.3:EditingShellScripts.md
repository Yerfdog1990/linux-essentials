## 11.3 Editing Shell Scripts

### 1. Overview of Text Editors in UNIX

UNIX systems provide many text editors, and choosing one often comes down to personal preference.

Two important editors mentioned in the LPI Essentials syllabus are:

* **GNU nano**

    * Simple and beginner-friendly
    * Ideal for editing small files like shell scripts
* **vi / vim (VI Improved)**

    * Very powerful and feature-rich
    * Has a **steep learning curve**

👉 In this section, the focus is on **nano** because it is easier to learn and use.

---

### 2. Opening a Script in Nano

To create or edit a shell script using nano:

```bash
nano test.sh
```

This opens the nano editor with a screen like:

```id="nano-screen-1"

#!/bin/sh

echo "Hello, World!"
echo -n "the time is "
```

---

### 3. Basic Navigation and Editing

Nano is very straightforward:

* Type normally to insert text
* Use **arrow keys** to move the cursor
* Use **Backspace/Delete** to remove text
* If working directly on a Linux machine, you can also use the **mouse** to:

    * Move the cursor
    * Highlight text

📌 The bottom of the screen shows available commands, which are **context-sensitive**.

---

### 4. Writing a Simple Shell Script in Nano

Inside nano, you can type a script like:

```bash id="script-example"
#!/bin/sh

echo "Hello, World!"
echo -n "the time is "
date
```

---

### 5. Exiting Nano

To exit nano:

👉 Press **Ctrl + X** (`^X`)

You will see:

```id="nano-exit"
Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?                 
 Y Yes                                                                       
 N No           ^C Cancel
```

Options:

* Press **Y** → Save changes
* Press **N** → Exit without saving
* Press **Ctrl + C** → Cancel

If you choose **Y**, press **Enter** to confirm the filename and save.

---

### 6. Saving Without Exiting

To save your work while staying in the editor:

👉 Press **Ctrl + O** (`^O`)

* You’ll be prompted to confirm the filename
* Press **Enter** to save
* You will remain inside nano

---

### 7. Editing Text Efficiently (Cut & Paste)

Nano provides simple shortcuts for editing:

#### Example Task:

Reorder lines in your script.

1. Move cursor to:

   ```
   echo -n "the time is "
   ```

2. Press:

    * **Ctrl + K** (twice) → Cuts two lines into buffer

3. Move cursor to desired location

4. Press:

    * **Ctrl + U** → Paste lines

👉 This allows you to rearrange code **without retyping**, improving efficiency.

---

### 8. Useful Nano Commands

| Command                         | Description                        |
| ------------------------------- | ---------------------------------- |
| **Ctrl + W**                    | Search text in the document        |
| **Ctrl + W**, then **Ctrl + R** | Search and replace                 |
| **Ctrl + G**                    | Display help menu (all commands)   |
| **Ctrl + Y / Ctrl + V**         | Page up / Page down                |
| **Ctrl + C**                    | Show cursor position and file size |

---

### 9. Key Takeaways

* Nano is a **simple and beginner-friendly text editor** for shell scripting.
* You can:

    * Open files with `nano filename`
    * Type and edit directly
    * Use keyboard shortcuts for efficiency
* Important commands:

    * **Ctrl + X** → Exit
    * **Ctrl + O** → Save
    * **Ctrl + K / Ctrl + U** → Cut and paste
* The command menu at the bottom is **dynamic and helpful for learning**

---

### 10. Practical Insight

Learning nano is important because:

* Shell scripts must be written in **plain text**
* You’ll frequently edit scripts directly in the terminal
* It’s fast, lightweight, and available on most Linux systems
