# 9.3 Archiving Files in Linux

When working with many files, compressing each file individually can reduce their size, but you would still have to manage **many separate files**.

For example, if you wanted to send 20 files to someone:

* Compressing them individually would produce **20 compressed files**.
* Managing or transferring them could still be inconvenient.

The solution to this problem is **archiving**.

---

# What is Archiving?

**Archiving** is the process of combining **multiple files and directories into a single file** called an **archive**.

An archive:

* Stores multiple files together
* Preserves directory structure
* Makes it easier to transfer or back up files

The traditional UNIX utility used for archiving is **`tar`**.

---

# The tar Command

The **`tar`** command stands for **TApe ARchive**.

Originally, it was designed to:

* Stream many files to **magnetic tape drives**
* Create backups
* Transfer large collections of files

Today it is widely used for:

* Backups
* Packaging software
* File distribution
* Data transfer

The `tar` command collects multiple files and stores them in a **single archive file**, which can later be **extracted back into the original files**.

These archive files are often called **tarballs**.

---

# Three Important tar Modes

The `tar` command operates in **three main modes**:

| Mode        | Purpose                        |
| ----------- | ------------------------------ |
| **Create**  | Create a new archive           |
| **List**    | Display contents of an archive |
| **Extract** | Extract files from an archive  |

Understanding these modes helps determine which **command-line options** to use.

---

# 9.3.1 Create Mode

Create mode is used to **build a new archive** from files or directories.

### Syntax

```bash
tar -c [-f ARCHIVE] [OPTIONS] [FILE...]
```

### Important Options

| Option       | Function                      |
| ------------ | ----------------------------- |
| `-c`         | Create a new archive          |
| `-f ARCHIVE` | Specify the archive file name |

The **archive name** follows the `-f` option, and the remaining arguments are the **files or directories to archive**.

---

## Example: Creating a Tar Archive

Suppose we want to archive all files that begin with **alpha**.

### Command

```bash
sysadmin@localhost:~/Documents$ tar -cf alpha_files.tar alpha*
```

### Check the archive

```bash
sysadmin@localhost:~/Documents$ ls -l alpha_files.tar
-rw-rw-r-- 1 sysadmin sysadmin 10240 Oct 31 17:07 alpha_files.tar
```

Explanation:

* `-c` → Create archive
* `-f alpha_files.tar` → Archive name
* `alpha*` → All files beginning with **alpha**

The result is a file called **alpha_files.tar**.

---

### Why the Archive Is Larger

Tar archives are often **slightly larger than the combined file sizes** because they store **metadata**, such as:

* File names
* Directory structure
* File permissions
* Ownership

This information allows tar to **recreate the original files exactly** when extracting.

---

# Compressing a Tar Archive

Tar files can also be **compressed** to reduce size.

One common method is to combine **tar with gzip compression**.

### Option for gzip Compression

| Option | Function                          |
| ------ | --------------------------------- |
| `-z`   | Compress or decompress using gzip |

---

## Example: Creating a Compressed Tar Archive

```bash
sysadmin@localhost:~/Documents$ tar -czf alpha_files.tar.gz alpha*
```

### Verify the file

```bash
sysadmin@localhost:~/Documents$ ls -l alpha_files.tar.gz
-rw-rw-r-- 1 sysadmin sysadmin 417 Oct 31 17:15 alpha_files.tar.gz
```

The compressed archive **alpha_files.tar.gz** is much smaller than the original **alpha_files.tar**.

---

## Viewing Compression Details

Since the file is compressed with gzip, you can view the compression statistics.

```bash
sysadmin@localhost:~/Documents$ gzip -l alpha_files.tar.gz
```

Output:

```
         compressed        uncompressed  ratio uncompressed_name
                417               10240  96.1% alpha_files.tar
```

Explanation:

* **compressed** → Size of compressed file
* **uncompressed** → Original archive size
* **ratio** → Compression percentage

---

# Common Archive File Extensions

Although extensions do not affect how the file works, there are common naming conventions.

| Archive Type        | Extension           |
| ------------------- | ------------------- |
| Tar archive         | `.tar`              |
| Gzip compressed tar | `.tar.gz` or `.tgz` |

---

# Using bzip2 Compression

Instead of gzip, tar can use **bzip2 compression**.

### Option

| Option | Function                           |
| ------ | ---------------------------------- |
| `-j`   | Compress or decompress using bzip2 |

### Example

```bash
sysadmin@localhost:~/Documents$ tar -cjf folders.tbz School
```

This command:

* Archives the **School directory**
* Compresses it with **bzip2**

---

### Common bzip2 Extensions

| Extension  |
| ---------- |
| `.tar.bz2` |
| `.tbz`     |
| `.tbz2`    |

---

# 9.3.2 List Mode

List mode displays the **contents of an archive without extracting it**.

### Syntax

```bash
tar -t [-f ARCHIVE] [OPTIONS]
```

### Important Options

| Option       | Function               |
| ------------ | ---------------------- |
| `-t`         | List contents          |
| `-f ARCHIVE` | Specify archive        |
| `-j`         | Decompress using bzip2 |

---

## Example: Listing Contents

```bash
sysadmin@localhost:~/Documents$ tar -tjf folders.tbz
```

Output:

```
School/
School/Engineering/
School/Engineering/hello.sh
School/Art/
School/Art/linux.txt
School/Math/
School/Math/numbers.txt
```

Explanation:

* `-t` → List contents
* `-j` → Use bzip2 decompression
* `-f folders.tbz` → Archive file

The archive stores **full directory paths**, so files remain in their correct folders when extracted.

---

# Using Pipelines to List Contents

The archive can also be processed using **pipes**.

Example:

```bash
sysadmin@localhost:~/Documents$ bunzip2 -c folders.tbz | tar -t
```

Output:

```
School/
School/Engineering/
School/Engineering/hello.sh
School/Art/
School/Art/linux.txt
School/Math/
School/Math/numbers.txt
```

Explanation:

* `bunzip2 -c folders.tbz` decompresses the archive
* `-c` sends the output to **standard output**
* The pipe `|` sends the data to **tar -t**
* `tar` reads from **standard input**

Pipelines allow commands to **work together efficiently**.

---

# 9.3.3 Extract Mode

Extract mode restores files from an archive.

### Syntax

```bash
tar -x [-f ARCHIVE] [OPTIONS]
```

### Important Options

| Option       | Function                |
| ------------ | ----------------------- |
| `-x`         | Extract files           |
| `-f ARCHIVE` | Specify archive         |
| `-j`         | Use bzip2 decompression |

---

## Preparing the Archive

Move the archive to a different directory.

```bash
sysadmin@localhost:~/Documents$ cd ~
sysadmin@localhost:~$ cp Documents/folders.tbz Downloads/folders.tbz
sysadmin@localhost:~$ cd Downloads
```

---

## Extract the Archive

```bash
sysadmin@localhost:~/Downloads$ tar -xjf folders.tbz
```

---

### Verify the extracted files

```bash
sysadmin@localhost:~/Downloads$ ls -l
```

Output:

```
total 8
drwx------ 5 sysadmin sysadmin 4096 Dec 20  2017 School
-rw-rw-r-- 1 sysadmin sysadmin  413 Oct 31 18:37 folders.tbz
```

Explanation:

* The archive remains unchanged
* A new directory **School** is created

---

### View the extracted directories

```bash
sysadmin@localhost:~/Downloads$ cd School
sysadmin@localhost:~/Downloads/School$ ls -l
```

Output:

```
total 12
drwx------ 2 sysadmin sysadmin 4096 Oct 31 17:45 Art
drwx------ 2 sysadmin sysadmin 4096 Oct 31 17:47 Engineering
drwx------ 2 sysadmin sysadmin 4096 Oct 31 17:46 Math
```

The original directory structure has been restored.

---

# Verbose Mode

Adding the **`-v` option** displays the files being processed.

| Option | Function       |
| ------ | -------------- |
| `-v`   | Verbose output |

---

### Example

```bash
sysadmin@localhost:~/Downloads$ tar -xjvf folders.tbz
```

Output:

```
School/
School/Engineering/
School/Engineering/hello.sh
School/Art/
School/Art/linux.txt
School/Math/
School/Math/numbers.txt
```

This helps monitor the extraction process.

---

# Important tar Option Order

The **`-f` option must come before the archive name**, because tar assumes the next argument is the file name.

Incorrect example:

```bash
sysadmin@localhost:~/Downloads$ tar -xjfv folders.tbz
```

Output:

```
tar (child): v: Cannot open: No such file or directory
tar (child): Error is not recoverable: exiting now
tar: Child returned status 2
tar: Error is not recoverable: exiting now
```

The error occurs because tar interprets **v as a file name**.

---

# Extracting Specific Files

You can extract only certain files from an archive.

Example:

```bash
sysadmin@localhost:~/Downloads$ tar -xjvf folders.tbz School/Art/linux.txt
```

Output:

```
School/Art/linux.txt
```

Only **one file** is extracted.

# Additional tar Features

The `tar` command also supports advanced operations, including:

* Extracting files using **patterns**
* **Excluding files** from archives
* **Displaying files to the screen** instead of writing them to disk
* Handling **large directory structures**

Full documentation for tar provides detailed explanations of these advanced features.

# Summary

**Archiving** allows multiple files and directories to be combined into a **single archive file**.

### Important tar Modes

| Mode    | Option | Purpose               |
| ------- | ------ | --------------------- |
| Create  | `-c`   | Create archive        |
| List    | `-t`   | View archive contents |
| Extract | `-x`   | Extract files         |

### Compression Options

| Option | Compression |
| ------ | ----------- |
| `-z`   | gzip        |
| `-j`   | bzip2       |

### Common Commands

Create archive:

```bash
tar -cf archive.tar files
```

Create compressed archive:

```bash
tar -czf archive.tar.gz files
```

List archive contents:

```bash
tar -tf archive.tar
```

Extract archive:

```bash
tar -xf archive.tar
```

The **tar command is one of the most important tools in Linux**, widely used for backups, software distribution, and transferring large collections of files efficiently.
