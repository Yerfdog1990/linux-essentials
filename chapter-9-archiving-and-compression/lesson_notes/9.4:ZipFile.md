# 9.4 ZIP Files in Linux

In **Microsoft Windows systems**, the most commonly used archiving format is the **ZIP file**. It is considered the **de facto standard** for compressing and archiving files in Windows.

Although ZIP files are **not as commonly used in Linux as tar archives**, Linux still provides strong support for them through two commands:

* `zip`
* `unzip`

These commands allow Linux users to **create, view, and extract ZIP archives**, making it easy to share files between **Linux and Windows systems**.

---

# Differences Between tar and zip

There is an important difference between how **tar** and **zip** operate.

With **tar and gzip/gunzip**, the same commands can often be used for **both compression and extraction** with different options.

However, with **zip and unzip**, each command has a **different purpose**, and **the same option may have different meanings** depending on which command is used.

Another difference is how filenames are handled:

* **tar** requires the `-f` option to specify the archive file.
* **zip and unzip** assume the first argument is the archive file, so the `-f` option is not required.

---

# Creating ZIP Archives

The **default behavior of the `zip` command** is to:

1. Create a new archive
2. Add files to it
3. Compress those files

---

## Syntax

```bash
zip [OPTIONS] [zipfile [file…]]
```

### Explanation

* **zipfile** → Name of the archive to create
* **file…** → Files to include in the archive

---

## Example: Creating a ZIP Archive

The following example creates an archive called **alpha_files.zip** containing files that start with **alpha**.

```bash
sysadmin@localhost:~/Documents$ zip alpha_files.zip alpha*
  adding: alpha-first.txt (deflated 32%)
  adding: alpha-second.txt (deflated 36%)
  adding: alpha-third.txt (deflated 48%)
  adding: alpha.txt (deflated 53%)
  adding: alpha_files.tar.gz (stored 0%)
```

### Explanation

* Each file added to the archive is displayed.
* The **compression ratio** is shown for each file.

Example meanings:

| Output         | Meaning                                  |
| -------------- | ---------------------------------------- |
| `deflated 32%` | File compressed by 32%                   |
| `stored 0%`    | File not compressed (already compressed) |

Files that are already compressed, such as `.gz` files, usually **cannot be compressed further**, so they are simply **stored**.

---

# Recursing Into Directories

Unlike the **tar command**, the `zip` command **does not automatically include files inside directories**.

If you simply add a directory name, zip will only include the **empty directory**.

To include all files and subdirectories, you must use the **recursive option**.

---

## Recursive Option

| Option | Function                                           |
| ------ | -------------------------------------------------- |
| `-r`   | Recursively include directories and their contents |

---

## Example: Archiving a Directory

```bash
sysadmin@localhost:~/Documents$ zip -r School.zip School
```

Output:

```
updating: School/ (stored 0%)
updating: School/Engineering/ (stored 0%)
updating: School/Engineering/hello.sh (deflated 88%)
updating: School/Art/ (stored 0%)
updating: School/Art/linux.txt (deflated 49%)
updating: School/Math/ (stored 0%)
updating: School/Math/numbers.txt (stored 0%)
  adding: School/Art/red.txt (deflated 33%)
  adding: School/Art/hidden.txt (deflated 1%)
  adding: School/Art/animals.txt (deflated 2%)
```

### Explanation

The output shows:

* Directories added to the archive
* Files being compressed
* Compression percentages

Because the `-r` option was used, **all files inside the School directory were included**.

# Listing the Contents of a ZIP Archive

The **`unzip` command** can list the files contained in a ZIP archive.

---

## List Option

| Option | Function                     |
| ------ | ---------------------------- |
| `-l`   | List contents of the archive |

---

## Example

```bash
sysadmin@localhost:~/Documents$ unzip -l School.zip
```

Output:

```
Archive:  School.zip
  Length      Date    Time    Name
---------  ---------- -----   ----
        0  2017-12-20 16:46   School/
        0  2018-10-31 17:47   School/Engineering/
      647  2018-10-31 17:47   School/Engineering/hello.sh
        0  2018-10-31 19:31   School/Art/
       83  2018-10-31 17:45   School/Art/linux.txt
        0  2018-10-31 17:46   School/Math/
       10  2018-10-31 17:46   School/Math/numbers.txt
       51  2018-10-31 19:31   School/Art/red.txt
       67  2018-10-31 19:30   School/Art/hidden.txt
       42  2018-10-31 19:31   School/Art/animals.txt
---------                     -------
      900                     10 files
```

### Explanation of Output Columns

| Column | Description                  |
| ------ | ---------------------------- |
| Length | File size                    |
| Date   | Last modification date       |
| Time   | Time modified                |
| Name   | File path inside the archive |

The output also shows:

* Total file count
* Total combined file size

# Extracting ZIP Files

The **default operation of the `unzip` command** is to **extract files from the archive**.

---

## Example: Extracting Files

```bash
sysadmin@localhost:~/Documents$ unzip School.zip
```

Example output:

```
Archive:  School.zip
replace School/Engineering/hello.sh? [y]es, [n]o, [A]ll, [N]one, [r]ename:
```

If files already exist, unzip will ask what to do:

| Option | Meaning           |
| ------ | ----------------- |
| y      | Replace this file |
| n      | Skip this file    |
| A      | Replace all files |
| N      | Replace none      |
| r      | Rename the file   |

---

# Extracting Files in a New Directory

To avoid overwrite prompts, extract the archive into a **different directory**.

---

## Step 1: Create a directory

```bash
sysadmin@localhost:~/Documents$ mkdir tmp
```

---

## Step 2: Copy the archive

```bash
sysadmin@localhost:~/Documents$ cp School.zip tmp/School.zip
```

---

## Step 3: Move to the directory

```bash
sysadmin@localhost:~/Documents$ cd tmp
```

---

## Step 4: Extract the archive

```bash
sysadmin@localhost:~/Documents/tmp$ unzip School.zip
```

Output:

```
Archive:  School.zip
   creating: School/
   creating: School/Engineering/
  inflating: School/Engineering/hello.sh
   creating: School/Art/
  inflating: School/Art/linux.txt
   creating: School/Math/
 extracting: School/Math/numbers.txt
  inflating: School/Art/red.txt
  inflating: School/Art/hidden.txt
  inflating: School/Art/animals.txt
```

### Explanation

* Directories are recreated
* Files are restored to their original locations

---

# Extracting Specific Files

Just like with `tar`, you can extract **specific files** from a ZIP archive.

---

## Attempt 1: Incorrect file path

```bash
sysadmin@localhost:~/Documents/tmp$ unzip School.zip linux.txt
```

Output:

```
Archive:  School.zip
caution: filename not matched: linux.txt
```

Explanation:

The file was **not found** because the archive stores the **full path**.

---

## Attempt 2: Correct file path

```bash
sysadmin@localhost:~/Documents/tmp$ unzip School.zip School/Math/numbers.txt
```

Output:

```
Archive:  School.zip
 extracting: School/Math/numbers.txt
```

Only that specific file is extracted.

---

# Extracting Files with Wildcards

You can also extract multiple files using **wildcards**.

Example:

```bash
sysadmin@localhost:~/Documents/tmp$ unzip School.zip School/Art/*t
```

Output:

```
Archive:  School.zip
  inflating: School/Art/linux.txt
  inflating: School/Art/red.txt
  inflating: School/Art/hidden.txt
  inflating: School/Art/animals.txt
```

This command extracts all files in **School/Art** that end with the letter **t**.

---

# Additional ZIP Features

The **zip** and **unzip** commands support many advanced features, including:

* Updating files inside archives
* Choosing different compression levels
* Extracting specific files
* Using **wildcards and patterns**
* Encrypting archives with passwords

More advanced options can be explored using the **man pages**:

```bash
man zip
man unzip
```

---

# Summary

ZIP archives are widely used, especially in **Windows environments**, but they are also fully supported in Linux.

### Important Commands

| Command | Purpose              |
| ------- | -------------------- |
| `zip`   | Create ZIP archives  |
| `unzip` | Extract ZIP archives |

### Common Options

| Option | Function                        |
| ------ | ------------------------------- |
| `-r`   | Recursively include directories |
| `-l`   | List contents of ZIP archive    |

### Example Commands

Create archive:

```bash
zip archive.zip files
```

Create archive with directories:

```bash
zip -r archive.zip directory
```

List archive contents:

```bash
unzip -l archive.zip
```

Extract archive:

```bash
unzip archive.zip
```

ZIP files provide a **simple and portable way to archive and compress files**, making them ideal for **sharing files between Linux and Windows systems**.
