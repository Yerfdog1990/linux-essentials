# 9.2 Compressing Files in Linux

**Compression** is the process of reducing the amount of data required to **store or transmit a file** while keeping it in a form that can later be **restored to its original state**.

Instead of storing repeated or unnecessary information, compression algorithms replace them with **shorter codes**, making the file smaller.

For example:

* Frequently used words in text may be replaced with smaller symbols.
* Large areas of the same color in an image may be represented with a simple code instead of storing each pixel.

The compressed file is usually **not used directly**. Instead, it must be **decompressed** before it can be read or used.

---

# Compression Algorithms

A **compression algorithm** is the method or procedure that a computer uses to compress a file.

Computer scientists continuously develop improved algorithms that:

* Compress files **more efficiently**
* Work **faster**
* Produce **smaller compressed files**

Different compression tools use different algorithms depending on their design goals.

---

# Types of Compression

Compression methods fall into **two major categories**:

1. **Lossless Compression**
2. **Lossy Compression**

---

## 1. Lossless Compression

In **lossless compression**, **no information is removed** from the original file.

This means:

* The file can be **compressed**
* Then **decompressed**
* And the result will be **identical to the original file**

This type of compression is required for files where **accuracy is critical**, such as:

* Documents
* Log files
* Software programs
* Databases
* Source code

Most Linux compression tools use **lossless compression**.

---

## 2. Lossy Compression

In **lossy compression**, **some data is intentionally removed** to reduce the file size.

When the file is decompressed, it will be **slightly different from the original**.

Example:

An image might contain two shades of green that look almost identical.
A lossy algorithm may treat both shades as **one color**, reducing the amount of stored data.

Even though information is lost, **human eyes or ears often cannot detect the difference**.

Lossy compression is commonly used for **multimedia files**, including:

* Images
* Music
* Videos

---

# Compression in Image Formats

Many image formats already use compression.

Examples include:

| Format   | Compression Type     |
| -------- | -------------------- |
| **JPEG** | Lossy compression    |
| **GIF**  | Lossless compression |
| **PNG**  | Lossless compression |

When saving images, users can often choose a **quality level**:

* **Lower quality**

    * Smaller file size
    * May show visual artifacts like rough edges or color distortion

* **Higher quality**

    * Looks closer to the original image
    * Larger file size

---

# Compressing Already Compressed Files

Compressing a file that is **already compressed** usually **does not make it smaller**.

This happens because compression algorithms rely on **patterns and repetition** in the data. Once those patterns are already encoded, the algorithm cannot reduce them further.

For example:

* Images stored as JPEG are already compressed.
* Compressing them again with another tool often **does not reduce their size significantly**.

With **lossless compression**, repeated compression and decompression does **not damage the file**.

However, repeatedly compressing and decompressing files using **lossy compression** can cause the file to **lose quality over time**, eventually becoming unusable.

---

# Compression Tools in Linux

Linux provides several utilities for compressing files.
The **most commonly used tool** is:

### gzip

The `gzip` command compresses files using a lossless compression algorithm.

---

# Example: Compressing a File with gzip

First, navigate to the directory containing the file.

### Step 1: Move to the Documents directory

```
sysadmin@localhost:~$ cd Documents
```

---

### Step 2: View the file details

```
sysadmin@localhost:~/Documents$ ls -l longfile*
-rw-r--r-- 1 sysadmin sysadmin 66540 Dec 20  2017 longfile.txt
```

Explanation:

* The file **longfile.txt** exists.
* Its size is **66540 bytes**.

---

### Step 3: Compress the file

```
sysadmin@localhost:~/Documents$ gzip longfile.txt
```

After running this command:

* The original file is **compressed**
* The original file **disappears**
* A compressed file is created with the extension **.gz**

---

### Step 4: Check the compressed file

```
sysadmin@localhost:~/Documents$ ls -l longfile*
-rw-r--r-- 1 sysadmin sysadmin 341 Dec 20  2017 longfile.txt.gz
```

Explanation:

* The new file is **longfile.txt.gz**
* Its size is now **341 bytes**

This shows a **huge reduction in size**, mainly because the original file contained **repetitive data**, which compression algorithms handle very efficiently.

---

# Viewing Compression Information

The `gzip` command can display information about compressed files using the **-l option**.

### Command

```
sysadmin@localhost:~/Documents$ gzip -l longfile.txt.gz
```

### Output

```
         compressed        uncompressed  ratio uncompressed_name
                341               66540  99.5% longfile.txt
```

Explanation:

* **compressed** → Size of the compressed file
* **uncompressed** → Original file size
* **ratio** → Compression ratio
* **uncompressed_name** → Name of the original file

In this example:

* Original file size = **66540 bytes**
* Compressed file size = **341 bytes**
* Compression ratio = **99.5% reduction**

---

# Decompressing Files

To restore a compressed file to its original state, it must be **decompressed**.

Two commands can perform this operation:

* `gunzip`
* `gzip -d`

Both commands achieve the same result.

---

## Using gunzip

### Command

```
sysadmin@localhost:~/Documents$ gunzip longfile.txt.gz
```

---

### Check the restored file

```
sysadmin@localhost:~/Documents$ ls -l longfile*
-rw-r--r-- 1 sysadmin sysadmin 66540 Dec 20  2017 longfile.txt
```

After decompression:

* The compressed file **longfile.txt.gz** disappears
* The original file **longfile.txt** is restored
* The file size returns to **66540 bytes**

---

# Consider This

The **gunzip** command is actually **a script that runs the gzip command with the correct parameters** to perform decompression.

---

# Other Linux Compression Tools

Linux also provides other compression utilities that work similarly to gzip.

These include:

### bzip2 and bunzip2

* `bzip2` compresses files
* `bunzip2` decompresses files
* Uses the **Burrows-Wheeler block sorting algorithm**
* Often produces **smaller compressed files than gzip**
* Requires **more CPU processing time**

Compressed files use extensions:

```
.bz
.bz2
```

---

### xz and unxz

Another modern compression tool is **xz**.

* `xz` compresses files
* `unxz` decompresses files

These utilities use the **Lempel-Ziv-Markov chain algorithm (LZMA)**.

Advantages:

* Excellent compression ratios
* Decompression speeds similar to gzip
* Often produces smaller files than both gzip and bzip2

Compressed files use the extension:

```
.xz
```

---

# Summary

Compression is a critical technique used to reduce file size for storage and transmission.

Key points:

### Compression Types

* **Lossless** → No data lost (used for documents, logs, software)
* **Lossy** → Some data removed (used for images, audio, video)

### Common Linux Compression Tools

| Tool  | Compression               | Extension  |
| ----- | ------------------------- | ---------- |
| gzip  | Lempel-Ziv algorithm      | .gz        |
| bzip2 | Burrows-Wheeler algorithm | .bz / .bz2 |
| xz    | LZMA algorithm            | .xz        |

### Important Commands

Compress a file:

```
gzip filename
```

View compression statistics:

```
gzip -l filename.gz
```

Decompress a file:

```
gunzip filename.gz
```

or

```
gzip -d filename.gz
```

Understanding compression tools is essential for managing **storage, backups, and file transfers in Linux systems**.
